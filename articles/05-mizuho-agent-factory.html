Amazon、Javaアプリ3万本のAI自動移行——Amazon Q Developerで作業4,500人年分を削減、日当〔1,000件操業

Amazon Web Services（AWS） | 2025年4月 | 読込中...

Amazonは2024年、社内で稼働する3万件以上のJavaアプリケーションをJava 17/21にアップグレードするマイグレーションプロジェクトに、自社開発のAIツール「Amazon Q Developer」を活用した。結果、これまでに要すると計算された4,500人年分の開発作業を削減し、ピーク時には1日当たり1,000件以上のアプリケーション移行を達成した。Amazon Q Developerの社内大規模導入は、コード移行・テスト自動化・IaC展開などの開発全工程でAIアシストが実用になることを示すランドマーク事例だ。

背景：Javaレガシーコードの山と「人材が足りない」問題

Amazonは社内に数万件規模のJavaアプリケーションを保有しており、その多くがJava 8・Java 11上で動作していた。メンテナンス・サポート期限切れのリスク、セキュリティパッチ応用の遅れ、新機能開発の遅延といった課題を抱えていた。

Java 17/21にアップグレードすることで、これらのリスクを解消できる。しかし問題は規模だった。Javaアップグレードはバージョン間の非互換変更への対応や接続ライブラリの履歴読み込みなど、エンジニアが手動で行う必要があった。社内の3万件を数年かかる手動作業で移行するのは現実的ではなかった。

Amazon Q Developerによる自動化アプローチ

Amazon Q Developer（旧: Amazon CodeWhisperer）は、AWSが2023年に発表した開発者向けAIアシスタントだ。コード生成・テスト自動化・脆弱性スキャンなどの機能に加え、「アプリケーション変換」機能を併せ持つ。Amazonはこの変換機能をJavaメダに特化した形で社内展開した。

具体的なプロセスは以下の通りだ。

  • 各Javaアプリの依存関係・API利用パターンをAmazon Qが分析
  • Java 17/21で廃止・変更されたAPIへの置換を自動特定
  • 移行候補コードをAIが自動生成、ビルド・テストを連続実行して検証
  • エラーがあればAIが再修正・再テストし、成功したら次のアプリへ

ピーク時には1日当たり1,000件以上のアプリケーション移行を達成。プロジェクト全体での移行処理数は最終的に3万件超を完了した。

Amazonの内部事例が示すのは、「AIによるコード変換」がコンセプトにとどまらず、本番級の大規模マイグレーション工事で実用になる段階に入ったということだ。レガシーコードの山を抱える中大規模開発組織にとって、Javaアップグレード以外にもテスト自動化・IaC化・ドキュメント整備などリソース消費の大きい工程でのAI活用を検討する価値がある。
